# Projekt Bazy Danych Systemu Krwiodawstwa

<h2>Opis Koncepcji Rozwiązania:</h2>

System krwiodawstwa jest kluczowym elementem infrastruktury zdrowotnej, zapewniającym dostępność krwi dla pacjentów potrzebujących transfuzji. 
Projekt bazy danych dla systemu krwiodawstwa ma na celu zarządzanie danymi pacjentów, dawców oraz banków krwi, a także śledzenie transakcji i testów krwi. 
Główne cele to zapewnienie integralności, dostępności i bezpieczeństwa danych oraz optymalizacja procesów związanych z krwiodawstwem.

**Projekt obejmuje następujące obszary:**

**Pacjenci:** Rejestracja i zarządzanie danymi pacjentów.
<br>
**Dawcy:** Rejestracja i zarządzanie danymi dawców.
<br>
**Banki Krwi:** Zarządzanie informacjami o bankach krwi.
<br>
**Transfuzje:** Śledzenie oddania i odbioru krwi.
<br>
**Badania krwii:** Przechowywanie wyników testów krwi.
<br>
**Pracownicy:** Zarządzanie personelem.
<br>
**Harmonogramy:** Planowanie terminów oddania krwi.
<br>
**Historia Pacjentów:** Przechowywanie historii transfuzji pacjentów.
<br>
**Grupy krwii:** Informacje o grupach krwii

# Wymagania i funkcje systemu

System musi umożliwiać wprowadzanie i aktualizację danych pacjentów i dawców.
System powinien przechowywać dane dotyczące banków krwi, takie jak nazwa, adres i dane kontaktowe.
System musi rejestrować wszystkie transakcje związane z oddaniem i odbiorem krwi.
System musi przechowywać wyniki testów krwi dla każdego dawcy.
System powinien przechowywać informacje o pracownikach banków krwi.
System musi umożliwiać planowanie wizyt dawców.
System musi przechowywać historię transfuzji dla każdego pacjenta.
System powinien zapewniać bezpieczeństwo danych poprzez dostęp do nich tylko dla osób upoważnionych.
Dawca i pacjent, powinni mieć zapewniony dostęp do ich historii krwiodawstwa.
Jako pracownik chcę mieć dostęp do danych dawców i pacjentów.
Jako pracownik chcę mieć dostęp do informacji danego banku krwii.
